# Reflections

> Longer-form analytical writing. When an observation has been thought about deeply enough to articulate a considered view.

---

## Purpose

Reflections are the output of sustained thinking. A reflection is written when:

- A pattern has been observed across enough data points to warrant careful analysis.
- A hypothesis is mature enough to be written out clearly.
- A connection between different parts of the framework deserves documentation.
- An important question has been partially answered.

Reflections are not casual. They take time to write. They represent the owner's best current thinking on a topic.

---

## Format (Suggested)

```markdown
# [Title of Reflection]

**Date:** YYYY-MM-DD
**Revised:** YYYY-MM-DD (if updated)
**Related Indicators:** [List]
**Related Case Studies:** [Links if applicable]

## The Observation
[What has been noticed that prompted this reflection]

## The Analysis
[Careful thinking about what it means]

## The Conclusion
[Current best understanding — held provisionally]

## Open Questions
[What remains uncertain]

## Revision History
[Brief notes on how thinking has changed over time]
```

---

## Naming Convention

Files should be named descriptively: `YYYY-MM-DD-topic.md`

Example: `2026-06-27-sofr-stress-pattern.md`
